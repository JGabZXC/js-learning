Dot vs Bracket Notation

Object

Example Code

                    
                        const gab = {
                            firstName: 'gab',
                            lastName: 'lopez',
                            age: 2024 - 2003
                        };

The Dot Notation

Dot Notation is a way to access the properties and methods of an object. It uses a period (dot) to connect the object name with the property or method name

Syntax

                        
                            objectName.propertyName

Example

                        
                            gab.firstName;

How Dot Notation Works

  • The dot is an operator that accesses the object and retrieves the property with the specified name.

The Bracket Notation

Bracket Notation can handle property names that are not valid JavaScript identifiers (like those with spaces, special characters, or reserved words). It also allows the use of variables to dynamically access properties.

Syntax

                        
                            objectName["propertyName"]

Example

                        
                            gab["firstName"]

Another Example

                        
                            const nameKey = 'Name';

                            console.log(gab["first" + nameKey]);
                            console.log(gab["last" + nameKey]);
How does it work
  • JavaScript will start executing first the code inside the bracket and it will create a string then it will go to gab object and retrieve that property

Adding Property to Object

Syntax

                        
                            gab.location = "Philippines"; // Dot Notation
                            gab["location"] = "Philippines"; // Bracket Notation

When do we use Bracket and Dot Notation?

Dot Notation

  1. Property Names are Valid JavaScript Identifiers:
    • The property name must start with a letter, underscore (_), or dollar sign ($) and can contain alphanumeric characters, underscores, and dollar signs.
    • Examples: obj.property, obj._property, obj.$property.

Bracket Notation

  1. Property Names Contain Spaces or Special Characters:
    • If a property name includes spaces, hyphens, or other special characters, you must use bracket notation.
  2. Property Names are Reserved Words:
    • If a property name is a reserved word in JavaScript, bracket notation is used to avoid syntax errors.
  3. Dynamic Property Names:
    • When the property name is stored in a variable or needs to be computed dynamically, bracket notation is necessary.
  4. Property Names are Not Known Until Runtime:
    • If the property names are determined at runtime, you need to use bracket notation.
